# 临港创新智算服务平台 - 多园区租户系统

这是一个临港创新智算服务平台的静态网页Demo，实现了多园区租户隔离和权限控制功能。

## 核心特性

### 多园区租户系统
- **7个独立园区**：科技城、影视基地、零界魔方、AI创新港、IC创新港、国际数据港、金融湾
- **租户隔离**：每个用户只能访问自己所属园区的页面
- **无公开注册**：系统不对外注册，账号由园区管理方分配
- **自动园区识别**：用户登录后系统自动识别所属园区并跳转到对应页面
- **内容同步**：创新生态和线下活动页面内容在所有园区保持同步
- **定制化展示**：每个园区有自己的主题色、特色模型、统计数据等

## 入口说明

本次系统有两个入口：

1. **login.html** - 系统主入口，用于已注册用户直接登录
2. **portal.html** - 园区展示页，展示平台接入的所有园区，可了解各园区信息

## 使用流程

### 方式一：直接登录（推荐入口）
1. 打开浏览器访问 `login.html`
2. 输入用户名和密码
3. 系统自动识别所属园区并跳转到该园区的主页

### 方式二：查看园区信息
1. 打开浏览器访问 `portal.html` 查看平台接入的所有园区
2. 点击"立即登录"进入登录页面
3. 登录后自动进入所属园区

**注意**：portal.html 纯展示页面，没有进入按钮，用户必须通过登录才能访问园区服务。

## 页面访问规则

- `login.html` - 登录页面（系统入口，无需登录）
- `portal.html` - 园区展示页（展示平台接入的所有园区，无需登录）
- `index.html?park=xxx` - 需要登录，且只能访问所属园区
- `models.html?park=xxx` - 需要登录，且只能访问所属园区
- `resources.html?park=xxx` - 需要登录，且只能访问所属园区
- `agent.html?park=xxx` - 需要登录，且只能访问所属园区
- `innovation.html?park=xxx` - 需要登录，所有园区内容同步
- `activities.html?park=xxx` - 需要登录，所有园区内容同步
- `dashboard.html?park=xxx` - 需要登录，且只能访问所属园区

## 权限控制机制

### 租户隔离
- 每个用户只能访问自己所属园区的页面
- 如果尝试访问其他园区的页面，会被阻止并提示权限错误
- 系统会自动将用户重定向到其所属园区的对应页面

### 登录验证
- 除 portal.html 外，所有页面都需要登录后才能访问
- 未登录用户会被自动重定向到 portal.html

### 功能开关
- 每个园区可以有自己的功能配置（models, resources, agent, innovation, activities）
-disabled 的功能在导航栏中会被隐藏

## 启动方式

### 直接打开
双击 `portal.html` 文件，在浏览器中打开即可查看。

### 使用本地服务器（推荐）
如果您希望获得更好的体验，可以使用本地服务器：

```bash
# 使用Python 3
cd demo
python -m http.server 8000

# 或使用Node.js的 http-server
npx http-server

# 或使用VSCode的 Live Server 插件
```

然后在浏览器访问 `http://localhost:8000/portal.html`

## 注意事项

1. **演示账户仅用于Demo测试**，实际部署时需要替换为真实的用户数据库
2. **密码安全性**：当前使用明文密码存储，生产环境应使用加密存储
3. **session存储**：当前使用 sessionStorage 存储登录状态，关闭浏览器后会清除
4. **URL参数**：园区通过 `park` 参数传递，如 `index.html?park=keji`

## 扩展说明

如需添加新的园区或用户：

1. 在 `config.js` 的 `PARK_CONFIG.parks` 中添加新的园区配置
2. 在 `auth.js` 的 `USER_ACCOUNTS` 中添加该园区的账户
3. 在 `portal.html` 中添加对应的园区卡片

## 开发建议

这是一个静态Demo，如果要将其转换为完整的生产应用，需要：

1. **后端API集成**：连接真实的模型API、计费系统、用户认证等
2. **数据库**：实现用户数据、账单、应用存储
3. **安全性**：添加CSRF保护、XSS防护、输入验证等
4. **性能优化**：代码压缩、懒加载、CDN缓存等
5. **构建工具**：使用Webpack、Vite等工具进行构建和打包
